CentOS解压版安装MySQL5.7.11

CentOS MySQL
发布日期:   2019-05-27
更新日期:   2021-03-30
文章字数:   986
阅读时长:   4 分
阅读次数:  

准备MySQL文件

将下载的安装包上传到服务器

或者使用wget http://dev.MySQL.com/get/Downloads/MySQL-5.7/mysql-5.7.11-Linux-glibc2.5-x86_64.tar.gz 命令在服务器在线下载,网速不好的情况不推荐,超级慢.

解压压缩包
tar -xvf mysql-5.7.11-Linux-glibc2.5-x86_64.tar.gz
修改文件夹名字
mv mysql-5.7.11-Linux-glibc2.5-x86_64.tar.gz mysql

新建data文件夹,用于存放数据

cd mysql
#data目录用于存放数据,也可以建在别的目录
mkdir data

新建用户和用户组

groupadd mysql

useradd -r -g mysql -s /bin/false mysql

chown -R root:root .

初始化数据库

./bin/mysqld --initialize --user=mysql --basedir=/data/mysql --datadir=/data/mysql/data
#执行完显示如下

#centos:/data/mysql #./bin/mysqld --initialize --user=mysql --basedir=/data/mysql --datadir=/data/mysql/data

# basedir: 就是你解压MySQL安装包路径,我这里是 /data/mysql
# datadir: 存放数据的路径, 我这里是 /data/mysql/data

bin/mysql_ssl_rsa_setup --datadir=/data/mysql/data
chown -R mysql:mysql /data/mysql

!!!注意这里初始化完在最后会给你 root 用户的一个默认密码,记住一下.

修改配置文件

${MySQL解压目录}/support-files 目录下面有个 my-default.cnf ,MySQL提供的默认配置文件

注意:MySQL5.7.18之后不再提供这个配置文件

cp support-files/my-default.cnf /etc/my.cnf

vim my.cnf

修改内容大致如下:

[client]
default-character-set = utf8mb4


[mysql]
default-character-set = utf8mb4


[mysqld]
character-set-server=utf8mb4
#版本5.6.19以后必须设置这一句才能使[client]中设置的编码有效
character-set-client-handshake = FALSE
collation-server=utf8mb4_unicode_ci
#表名是否区分大小写：1表示不区分大小写，2表示区分
lower_case_table_names=1

basedir = /data/mysql
datadir = /data/mysql/data


port = 3306
socket = /tmp/mysql.sock

注意:这里的 basedir 和 datadir 的路径跟上面初始化数据库的地址需要一致,不然会报错

将MySQL加入服务

cp support-files/mysql.server /etc/init.d/mysql

添加开机启动mysql服务

chkconfig --add mysql
chkconfig --level 2345 mysql on

启停MySQL服务

service mysql start|stop|restart|status

登录mysql修改默认密码

./bin/mysql -uroot -p
会提示输入上面初始化数据库时给你的默认密码

修改密码:

mysql>set password = 'yourpassword';

为了方便，可以为/data/mysql/bin/mysql添加软链接到/usr/bin下，这样就可以在任意路径下使用mysql命令了：
ln -fs /data/mysql/bin/mysql /usr/bin/mysql

无法远程连接的解决办法

修改mysql数据库中的user表使相应的用户能从某一主机登陆

mysql -u root –p
mysql>use mysql;
mysql>update user set host = '%' where user = 'root';
mysql>select host, user from user;

没有授予相应的权限

例如，你想root使用123456从任何主机连接到mysql服务器。
mysql>GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' IDENTIFIED BY '123456' WITH GRANT OPTION;

123456 是连接时输入的密码.

忘记密码需要重置

修改 my.cnf 文件
vim my.cnf
在 [mysqld] 下面加入 skip-grant-tables

保存退出
重启MySQL服务
service mysql restart
重启之后输入#mysql即可进入mysql。
修改密码

mysql> use mysql;
mysql> update user set password=password("你的新密码") where user="root";
mysql> flush privileges;
mysql> quit

遇到的一些错误

  1. Starting MySQL. ERROR! The server quit without updating PID file (/xxx/centos.novalocal.pid).

    看一下 xxx 目录所有者是不是mysql

  2. 初始化数据库显示 ./bin/mysqld: error while loading shared libraries: libaio.so.1: cannot open shared object file: No such file or directory

yum -y install numactl
yum install  libaio-devel.x86_64

执行完这两个就ok了
